# 🧮 MathBuddy Agent Architecture Notes

How I built a friendly, adaptive K-12 math tutoring agent using Streamlit and LangChain.

---

## 🧠 Key Technical Concepts

### 1. Streamlit Session State (`st.session_state`)
* **The Problem:** Web apps are inherently stateless—they "forget" everything every time the screen updates or a user sends a message.
* **The Solution:** We use `st.session_state` to create a persistent memory cache. This allows the AI to remember earlier parts of the conversation context.

### 2. LangChain Message Object Types
Professional conversational agents rely on explicit message types to structure the data sent to the LLM:
* `SystemMessage`: Establishes the background rules, safety guardrails, and persona (e.g., *MathBuddy's friendly persona*).
* `HumanMessage`: Represents the query typed by the student.
* `AIMessage`: Represents the response generated by the assistant.

### 3. State-Based Authentication
* We implement a conditional gateway using `st.session_state["authenticated"]`. 
* This ensures that the LLM logic and API keys are only initialized after a successful mock login through Google or Microsoft.

### 4. Native LaTeX Formatting
* Streamlit natively converts standard text into beautiful mathematical rendering.
* By ensuring the LLM wraps math logic inside dollar signs (e.g., `$f(x) = x^2$`), formulas display beautifully to the user.
